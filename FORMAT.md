# TopoDB on-disk format (v3)

This is the durable contract for a TopoDB redb file. Code wins if it differs
from this document. `crates/topodb/tests/fixtures/v1.redb` and
`crates/topodb/tests/fixtures/v2.redb` are FROZEN migration corpora, plus a
larger frozen v2 corpus (`v2-workload.redb`) used by `migrate_v3.rs`'s own
migration unit tests — none of the three is regenerable by this build (see
`format_fixture.rs`'s module doc comment) and none may be deleted or
byte-modified. `crates/topodb/tests/fixtures/v3.redb` is the native v3
fixture, regenerated by `format_fixture.rs::regenerate_v3_fixture` and
exercised by `v1_fixture_opens_and_reads` / `v2_fixture_opens_and_reads` /
`v3_fixture_opens_and_reads`.

## Version and migration

`storage::FORMAT_VERSION` is **3**. `Storage::open_with_options` opens (or
creates) every table, reads META `"format_version"`, and dispatches on it —
all inside ONE redb write transaction, committed once at the very end:

| stored version | on open |
|---|---|
| absent (brand-new file) | stamp `FORMAT_VERSION` (3) directly — `storage.rs: Storage::open_with_options`, `None` arm |
| 3 | open directly, no rewrite — `Some(3) => {}` |
| 2 | `migrate_v3::migrate_v2_to_v3`, then stamp 3 — `Some(2)` arm |
| 1 | `migrate::migrate_v1_to_v2` (stamps 2), immediately followed by `migrate_v3::migrate_v2_to_v3` (stamps 3) — `Some(1)` arm, chained in one call |
| > 3 | `TopoError::UnsupportedFormat { found, supported: 3 }` — `Some(found) if found > FORMAT_VERSION` |
| anything else (currently only 0, which can't occur since real versions start at 1) | `TopoError::Encoding` — `Some(found)` catch-all arm |

Because table creation, migration, and the version stamp all happen inside the
single write transaction `open_with_options` opens at its top and commits once
at the bottom (`storage.rs`, around `let tx = s.db.begin_write()...` through
`tx.commit()`), a crash or kill at any point before that one commit leaves the
file at exactly its prior on-disk version — including mid-way through the
chained v1→v2→v3 path. There is no partially-migrated state observable on
reopen.

Migration rewrites are scoped precisely:
- **v1 → v2** (`migrate::migrate_v1_to_v2`, frozen): decodes the old
  embedded-embedding `NodeRecordV1`/`EdgeRecordV1` rows, externalizes
  embeddings into `EMBEDDINGS` (still ULID-keyed at this point), interns
  label/edge-type/prop-key strings into `DICT`, and rewrites `NODES`/`EDGES`
  via `disk::node_to_disk`/`disk::edge_to_disk` (the frozen v2 ULID-keyed
  encode shape). Never touches `OPS`, `POSTINGS`/`FTS_DOCS`/`FTS_STATS`,
  `COUNTERS`, or unrelated `META` keys.
- **v2 → v3** (`migrate_v3::migrate_v2_to_v3`): consumes v2 `NODES`/`EDGES`
  (decoded through the frozen `NodeRecordDiskV2`/`EdgeRecordDiskV2` twins) plus
  a pre-drain snapshot of the old ULID-keyed `EMBEDDINGS`/`COUNTERS` rows, and
  produces: dense node/edge slots (`NODE_SLOTS`/`NODE_IDS`/`EDGE_SLOTS`/
  `EDGE_IDS`), slot-rekeyed `NODES`/`EDGES`/`EMBEDDINGS`/`COUNTERS`, a seeded
  `SCOPES` registry, `OUT_ADJ`/`IN_ADJ` adjacency built from every edge row,
  `PROP_INDEX` rebuilt from the caller's `IndexSpec`, and `POSTINGS`/
  `FTS_DOCS`/`FTS_STATS` fully **rebuilt** (not transcoded) via the same
  `fts_update` the write path uses — the pre-W2b v2 FTS layout is
  byte-incompatible with v3's scope-id/slot keying, so it is drained and
  regenerated once per node instead. `DICT` rows carry forward unchanged
  (label/type/prop-key ids are stable across v2→v3). `OPS` is untouched;
  `META` is untouched except `"next_node_slot"`/`"next_edge_slot"`, which this
  step clears and then repopulates as it allocates fresh slots (a v2 file has
  no slot counters yet — slots don't exist before v3).

Redb free space means the on-disk file size need not shrink even where
logical row bytes do — e.g. a v1 file's per-node embedded embeddings become
smaller externalized rows after migration, but the file itself may stay the
same size or grow, since redb reuses/retains pages via its own free list
rather than truncating. Explicit compaction (`redb::Database::compact`, used
by `regenerate_v3_fixture` when shipping a fixture) is a separate, deliberate
step never run automatically by `open_with`.

## Tables

`storage.rs: Storage::open_with_options` unconditionally opens **eighteen**
tables on every open (verified from its `existing` block, `OPS` through
`META`) — this is the complete set:

| table | key | value | defined in |
|---|---|---|---|
| `ops` | u64 sequence (native redb `u64` key type — LE bytes, but ordered numerically by redb's own `Key::compare`, not lexicographically) | raw postcard `Op` (never interned or framed) | `storage.rs: OPS` |
| `meta` | UTF-8 string | key-specific bytes (see below) | `storage.rs: META` |
| `nodes` | 8-byte BE node slot | framed postcard `disk::NodeRecordDiskV3` | `storage.rs: NODES` |
| `edges` | 8-byte BE edge slot | framed postcard `disk::EdgeRecordDiskV3` | `storage.rs: EDGES` |
| `embeddings` | 8-byte BE node slot | framed postcard `(String, Vec<f32>)` | `storage.rs: EMBEDDINGS` |
| `counters` | 8-byte BE node slot | plain postcard `counters::AccessStats` (not framed) | `storage.rs: COUNTERS` |
| `postings` | `scope_id (4-byte BE) ++ term` UTF-8 | framed delta-varint `(slot_delta, tf)` pairs | `storage.rs: POSTINGS`, `fts.rs` |
| `fts_docs` | 8-byte BE node slot | plain postcard `u32` token length (not framed) | `storage.rs: FTS_DOCS`, `fts.rs` |
| `fts_stats` | 4-byte BE scope id | plain postcard `(u64, u64)` = `(doc_count, total_len)` (not framed) | `storage.rs: FTS_STATS`, `fts.rs` |
| `dict` | `[kind:u8][id:u32 BE]` (5 bytes) | UTF-8 interned string (not framed) | `dict.rs: DICT` |
| `scopes` | 4-byte BE scope id | 17-byte `scope_key` | `scopes.rs: SCOPES` |
| `node_slots` | 16-byte BE node ULID | 8-byte LE slot | `slots.rs: NODE_SLOTS` |
| `node_ids` | 8-byte BE node slot | 16-byte BE node ULID | `slots.rs: NODE_IDS` |
| `edge_slots` | 16-byte BE edge ULID | 8-byte LE slot | `slots.rs: EDGE_SLOTS` |
| `edge_ids` | 8-byte BE edge slot | 16-byte BE edge ULID | `slots.rs: EDGE_IDS` |
| `out_adj` | `[slot:8][edge_type:4][chunk:4]` (16 bytes) | framed adjacency block | `adj.rs: OUT_ADJ` |
| `in_adj` | `[slot:8][edge_type:4][chunk:4]` (16 bytes) | framed adjacency block | `adj.rs: IN_ADJ` |
| `prop_index` | `[prop_key:4][tag:1][canonical value][slot:8]` | empty (existence-only key) | `prop_index.rs: PROP_INDEX` |

`node_key`/`edge_key` (16-byte BE ULID keys, `storage.rs`) are NOT part of the
live v3 read/write path — they remain in use only by `migrate.rs`'s frozen
v1→v2 step and by `migrate_v3.rs`, which reads OLD v2-keyed
`EMBEDDINGS`/`COUNTERS` rows by ULID before re-keying them to slots.

## Key and value encodings

**Dense slots** (`slots.rs`): `NODE_SLOTS`/`EDGE_SLOTS` map a 16-byte BE ULID
key (`id_key`) to an 8-byte **LE** slot value; `NODE_IDS`/`EDGE_IDS` are the
reverse mapping, 8-byte BE slot key (`slot_key`) to 16-byte BE ULID value.
Allocation (`slots.rs: alloc`) is monotonic and never reuses a freed slot:
`META` counters `"next_node_slot"`/`"next_edge_slot"` (8-byte LE u64) hold the
next slot to hand out. `NODES`/`EDGES`/`EMBEDDINGS`/`COUNTERS`/`FTS_DOCS` are
all keyed by the 8-byte BE slot (`storage.rs: slot_key`).

**Dictionary** (`dict.rs`): key is `[kind:u8][id:u32 BE]` (5 bytes,
`dict::key`); value is the raw UTF-8 interned string (not framed). The kind
byte is an **APPEND-ONLY** registry (`dict::DictKind`): `0x00` Label, `0x01`
EdgeType, `0x02` PropKey. Dictionary rows are derived, append-only state —
`intern` never removes or renumbers an id. An id absent from `Dicts::resolve`
while decoding is corruption and returns `TopoError::Encoding`.

**Scopes** (`scopes.rs`): key is a 4-byte BE `u32` registry id; value is the
fixed-width 17-byte `scope_key` (`storage.rs: scope_key`) — 1-byte tag
(`0x00` Shared, all-zero remainder; `0x01` Id, followed by the 16-byte BE
scope ULID). Row `0` is always `Scope::Shared`, seeded by `scopes::seed_shared`
on every `open_with` and by every rebuild/migration path;
`ScopeRegistry::load_table_for_rebuild` refuses to load a table where id `0`
isn't `Shared`. Interning is append-only, same doctrine as the dictionary.

**Value frame** (`codec.rs`): every `nodes`, `edges`, `embeddings`,
`postings`, and adjacency (`out_adj`/`in_adj`) block value is wrapped in a
1-byte codec-tag frame (`codec::frame_value`/`unframe_value`) — verified by
grepping every `frame_value`/`unframe_value` call site in `src/`. `counters`,
`fts_docs`, `fts_stats`, `dict`, `scopes`, and `prop_index` are plain
(unframed) postcard or raw bytes; `ops` and `meta` are untouched raw
postcard/bytes. The codec byte is an **APPEND-ONLY** registry: `0x00`
(`CODEC_RAW`) uncompressed postcard; `0x01` unused (skipped in the numbering,
available for a future codec); `0x02` (`CODEC_LZ4`) an `lz4_flex`
size-prepended block. `frame_value` only attempts lz4 for raw payloads `>=
512` bytes, and keeps the compressed form only if it's `<= 90%` of the raw
size — otherwise it falls back to raw. `unframe_value` rejects an lz4 frame
whose declared decompressed size exceeds `256 MiB`. `lz4_flex` is pure Rust;
no C zstd dependency is used.

**Adjacency** (`adj.rs`): key is `[slot:8 BE][edge_type:4 BE][chunk:4 BE]`
(16 bytes, `adj::out_adj_key`) — `(slot, edge_type)` addresses a bounded range
of chunks, `chunk` is a dense split counter (`adj::CHUNK_SPLIT_TARGET` = 8 KiB
encoded-size target per chunk; `adj_insert` splits a chunk in half once it
would exceed that). The framed block payload (`adj::encode_block`/
`decode_block`) is:
1. `block_format: u8` — **APPEND-ONLY** registry, currently only `0x00`
   (`ADJ_BLOCK_FORMAT_V0`); any other byte is rejected as
   `"unknown adjacency block format"`.
2. `count: varint`
3. `count` entries, each: `target_delta: varint` (delta from the previous
   entry's `target`, first entry relative to 0 — entries are required
   target-sorted, `encode_block` rejects an out-of-order slice), `edge: varint`
   (edge slot), `scope: varint` (scope registry id, cast from `u32`),
   `valid_from: varint` (zigzag-encoded `i64`), then a 1-byte `valid_to` tag
   (`0` = `None`; `1` = `Some`, followed by a zigzag `varint`).

**Prop index** (`prop_index.rs`): key is `[prop_key:4 BE][tag:1][canonical
value bytes][slot:8 BE]` (`prop_index::index_key`); value is empty
(existence-only row). The value tag registry (`prop_index::value_bytes`,
fixed — not one of the three append-only byte registries called out in the
Evolution policy below, but stable in practice since it mirrors
`index::IndexValue`'s variants): `0` Str (raw UTF-8 bytes), `1` Int (sign-flip
trick: `(v as u64) ^ (1 << 63)`, BE 8 bytes — preserves signed ordering in an
unsigned lexicographic key space), `2` Bool (1 byte, `0`/`1`), `3` Bytes (raw),
`4` DateTime (same sign-flip as Int, BE 8 bytes). `PropValue::Float` is never
indexed (`IndexValue::of` returns `None` for it). `lookup` range-scans the
`[prop_key][tag][value]` prefix and then filters by exact key length
(`prefix.len() + 8`) — load-bearing, since a longer Str/Bytes value sharing a
shorter one's byte prefix (`"ab"` vs `"abc"`) would otherwise fall inside the
same range scan.

**Postings** (`fts.rs`): key is `scope_id.to_be_bytes() (4 bytes) ++ term`
UTF-8 bytes (`fts::posting_key`) — the fixed-width scope prefix means no term
can straddle two scopes' key ranges even when their BE bytes share a leading
byte (e.g. scope `1` = `00 00 00 01` vs scope `256` = `00 00 01 00`, pinned by
a dedicated test). The framed value (`fts::encode_postings`/`decode_postings`)
is `count: varint` followed by `count` `(slot_delta: varint, tf: varint)`
pairs, ascending by node slot, first delta relative to 0. `fts_docs` (node
slot → plain postcard `u32` token count) and `fts_stats` (4-byte BE scope id →
plain postcard `(doc_count: u64, total_len: u64)`) are unframed and maintained
by the same `fts::fts_update` used by `apply_batch`, `rebuild_state_from_ops`,
`ensure_index_spec`'s reindex, and `migrate_v3::migrate_v2_to_v3`. A scope
whose last document is removed drops its `fts_stats` row entirely (same
empty-key doctrine as the prop index and postings).

**META keys** (`storage.rs`, `slots.rs`): `"format_version"` (4-byte LE
`u32`); `"oldest_seq"` (8-byte LE `u64`, absent ⇒ 1, written only by
`compact_ops_through`); `"index_spec"` (postcard `IndexSpec`, both lists
sorted by `(label, prop)` before encoding so declaration order never looks
like a spec change); `"next_node_slot"`/`"next_edge_slot"` (8-byte LE `u64`
monotonic slot counters, `slots.rs`). Legacy Plan-2 keys `"fts_spec"`,
`"fts_doc_count"`, `"fts_total_len"` are read (and, if present, removed) only
by `ensure_index_spec` when opening a Plan-2 file (a file with the legacy
`"fts_spec"` key present — a Plan-1 file has neither `"fts_spec"` nor
`"index_spec"` and never triggers the removal) — no live write path writes
them.

## Records

`disk::NodeRecordDiskV3 { id: NodeId, scope: u32, label: u32, props:
BTreeMap<u32, PropValue> }` and `disk::EdgeRecordDiskV3 { id: EdgeId, scope:
u32, ty: u32, from: u64, to: u64, props: BTreeMap<u32, PropValue>, valid_from:
i64, valid_to: Option<i64> }` are the live v3 record-table shapes
(`disk.rs`): `scope` is the interned `ScopeRegistry` id, edge `from`/`to` are
node **slots** (not ULIDs), and label/edge-type/prop-key strings are
`Dicts`-interned `u32`s. Each row still carries its own ULID `id` — even
though the table is slot-keyed — so no extra table hop is needed to answer
"what ULID is this row." `disk::node_to_disk_v3`/`node_from_disk_v3` and
`edge_to_disk_v3`/`edge_from_disk_v3` convert to/from the public,
string-carrying `state::NodeRecord`/`EdgeRecord`, which are unchanged by the
v3 cutover. `disk::NodeRecordDisk`/`EdgeRecordDisk` (no `V3` suffix) are a
SEPARATE, frozen v2 ULID-keyed encode shape kept only because `migrate.rs`'s
v1→v2 step calls them; nothing in the live v3 path reads or writes them.

Node disk rows carry no embedding: `storage.rs: read_node`/`read_node_by_slot`
join the cold `EMBEDDINGS` row back onto the record by slot before returning
it, so an ordinary property edit never rewrites a node's (potentially large)
vector.

## The `ops` table (unchanged since v1)

`OPS` is a u64-sequence-keyed table of raw postcard `Op` values (`op.rs`) —
never interned through `DICT`, never wrapped in the `codec.rs` value frame.
This is deliberate: the op log is self-describing, so replay
(`Storage::rebuild_state_from_ops`) never depends on what dictionary ids,
scope ids, or slots happened to be assigned when the log was first written.
Every other table in this document — dictionaries, the scope registry, slots,
adjacency, the prop index, and the FTS tables — is *derived* state,
reconstructible from `OPS` plus (for v1/v2 files) a one-time migration.

## Corruption contract

`storage.rs: read_node`/`read_edge` (and their `_by_slot` siblings' callers)
distinguish two failure shapes: no `NODE_SLOTS`/`EDGE_SLOTS` mapping at all is
an ordinary `Ok(None)` ("this id never existed"); a mapping that resolves to a
slot whose `NODES`/`EDGES` row is *absent* is `TopoError::Encoding` — never
silently treated as "not found." The mapping and its record row are written
and removed atomically on every code path, so they can only diverge if the
file itself is damaged, and that must surface loudly.

## Rebuild determinism

`Storage::rebuild_state_from_ops` drains `NODES`, `EDGES`, `EMBEDDINGS`,
`DICT`, `SCOPES`, `NODE_SLOTS`, `NODE_IDS`, `EDGE_SLOTS`, `EDGE_IDS`,
`OUT_ADJ`, `IN_ADJ`, `PROP_INDEX`, `POSTINGS`, `FTS_DOCS`, and `FTS_STATS` —
every table this document calls *derived* — then replays `OPS` from seq 1
through the same `apply_op` the normal write path (`apply_batch`) uses, so
there is no parallel mutation logic to drift out of sync. `COUNTERS` is
deliberately left untouched: access statistics are auxiliary telemetry outside
the op log, never appended to `OPS`, so a rebuild must preserve them rather
than reset them to zero. Refuses with `TopoError::Compacted { oldest }` once
`oldest_seq > 1`, since a compacted log is no longer a full history and replay
from genesis is impossible by definition.

## Evolution policy

On-disk serde enum variants (`Op`, `PropValue`, `IndexValue`, ...) are
append-only. In addition, three plain byte registries are **APPEND-ONLY** —
an existing value's meaning may never be repurposed, only new values added:

- `dict.rs: DictKind` — `0x00` Label, `0x01` EdgeType, `0x02` PropKey.
- `codec.rs` value frame — `0x00` raw postcard, `0x01` unused/reserved,
  `0x02` lz4.
- `adj.rs: ADJ_BLOCK_FORMAT_V0` — currently the only defined adjacency block
  format byte (`0x00`); `decode_block` rejects any other value.

The op log deliberately retains full strings and no frame so replay stays
independent of dictionary/scope-registry assignment order (see "The `ops`
table," above). `Storage::rebuild_state_from_ops` drains and regenerates every
derived v3 table — dictionaries, scope registry, slots, adjacency, prop index,
and FTS — while replaying this self-describing log (see "Rebuild
determinism," above).
